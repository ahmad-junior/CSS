Selectors

Combinators

Combinators are used to combine two or more selectors. There are four types of combinators in CSS:

  • Descendant combinator

    • Syntax

                    
                    selector1 selector2 {
                        property: value;
                    }
  • Child combinator

    • Syntax

                    
                    selector1 > selector2 {
                        property: value;
                    }
  • Adjacent sibling combinator

    • Syntax

                    
                    selector1 + selector2 {
                        property: value;
                    }
  • General sibling combinator

    • Syntax

                    
                    selector1 ~ selector2 {
                        property: value;
                    }

Pseudo Class

Pseudo-classes are used to define special states of an element. For example, it can be used to:

  • Style an element when a user mouses over it
  • Style visited and unvisited links differently
  • Style an element when it gets focus

There are two types of pseudo-classes:

  • Link pseudo-classes
  • UI element states

Link pseudo-classes are used to define a special state of links. There are four link pseudo-classes:

  • a:link - Selects all unvisited links
  • a:visited - Selects all visited links
  • a:hover - Selects the link the mouse pointer is over
  • a:active - Selects the active link

UI element states are used to define a special state of user interface (UI) elements. There are eight UI element states:

  • :hover - Selects elements when you mouse over them
  • :active - Selects the active element
  • :focus - Selects the element that has focus
  • :target - Selects the current active element (clicked on a URL containing a reference to that element)
  • :lang() - Selects elements based on the language of the element
  • :enabled - Selects all enabled elements
  • :disabled - Selects all disabled elements
  • :checked - Selects all checked elements

Pseudo Eelement

Pseudo-elements are used to style specified parts of an element. For example, it can be used to:

  • Insert content before, after, or in-between the content of an element
  • Style the first letter, or line, of an element

There are two types of pseudo-elements:

  • Content pseudo-elements
  • Text pseudo-elements

Content pseudo-elements are used to insert content before, after, or in-between the content of an element. There are two content pseudo-elements:

  • ::before - Inserts content before the content of an element
  • ::after - Inserts content after the content of an element

Text pseudo-elements are used to style the first letter, or line, of an element. There are two text pseudo-elements:

  • ::first-letter - Selects the first letter of an element
  • ::first-line - Selects the first line of an element

Attr Selectors

Attribute selectors are used to select elements with a specific attribute or attribute value.

There are three types of attribute selectors:

  • [attribute] - Selects all elements with the specified attribute
  • [attribute=value] - Selects all elements with the specified attribute value
  • [attribute~=value] - Selects all elements with the specified attribute value, separated by spaces

Specificity

Specificity is used to determine which style rule should be applied if more than one rule matches a certain element.

There are four types of selectors, each with a different specificity:

  • Inline style - 1000
  • IDs - 100
  • Classes, pseudo-classes, attributes - 10
  • Elements, pseudo-elements - 1

Specificity is calculated by adding the specificity of each selector in a rule. For example, the following rule has a specificity of 0,0,1,1 (0+0+1+1):

            
                p {
                    color: red;
                }

Inline styles have the highest specificity, IDs have a specificity of 100, classes, pseudo-classes, and attributes have a specificity of 10, and elements and pseudo-elements have a specificity of 1.

If two or more rules have the same specificity, the last declared rule will be applied.

Specificity can be used to override styles. For example, the following rule has a specificity of 0,0,0,2 (0+0+0+2):

            
                * {
                    color: red;
                }

Since the rule has a specificity of 0,0,0,2, it will override all other rules.
